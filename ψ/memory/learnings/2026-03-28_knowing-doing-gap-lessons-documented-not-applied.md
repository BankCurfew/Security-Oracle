# Knowing-Doing Gap: Lessons Documented but Not Applied

**Date**: 2026-03-28
**Source**: Self-improvement check — session 7
**Tags**: meta-process, discipline, self-awareness

## Pattern

The same lessons have been written multiple times without being acted on:

- **Mar 22**: "Must commit all untracked files at session end" — 20+ files uncommitted
- **Mar 27**: "Retro discipline must be per-session" — 5-day gap flagged
- **Mar 28**: Still 31 files uncommitted. Same lesson, third occurrence.

Additionally, the Mar 27 learning about "CRITICAL findings need active follow-up" was written but the ElevenLabs key rotation (CRITICAL) still has no tracking mechanism — just a markdown handoff.

Writing a learning file is not the same as changing behavior. If the same lesson appears twice, the problem isn't awareness — it's execution.

## Lesson

A learning that doesn't change the next session's behavior is just a journal entry. Before writing a new learning, check: **is this the same lesson I already wrote?** If yes, the real problem is the execution barrier, not the insight.

## Application

1. When a learning repeats, don't write it again — instead identify and remove the execution barrier (e.g., "why didn't I commit? Was I out of context? Was there no reminder?")
2. Convert recurring learnings into hard rules with mechanical triggers — if "commit at session end" keeps failing, it needs a hook or checklist, not another markdown file
3. Before ending any session: `git status` — if untracked/uncommitted files exist, commit them. No exceptions, no "next session."
4. For CRITICAL findings: create an oracle thread or issue, not just a handoff file. Markdown has no enforcement.
